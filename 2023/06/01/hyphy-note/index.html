hyphy_note

上个星期和一位老师交流，老师问我有没有用过hyphy这个软件，我当时说不知道，所以赶紧回来学习一下。

Hyphy - Hypothesis testing using Phylogenies

Hyphy is an open-source software package for the analysis of genetic sequences using techniques in phylogenetics, molecular evolution, and machine learning. It features a complete graphical user interface (GUI) and a rich scripting language for limitless customization of analyses. Additionally, Hyphy features support for parallel computing environments (via message passing interface (MPI)) and it can be complied as a shared library and called from other programming environments such as Python and R. Hyphy is the computational backbone powering datamonkey.org.

虽然github介绍有GUI界面但是GUI界面版本的Hyphy已经不在维护，要么使用CLI，或者使用在线网站datamonkey。

首先正选择分析一般使用PAML，Easycodeml,hyphy分析.

以PAML为例 会使用三种模型

site 模型，branch模型,branch-site模型，我刚大致看了几篇文章，和预期的类似一般会选择branch-site模型。之所以不简单使用dn/ds计算是因为这种方式十分严格。

某些文献称：

branch模型鉴定出来的基因成为快速进化基因

branch-site模型鉴定出来的基因成为正选择基因

但是我发现hyphy的aBSREL(adaptive brach-site random effects likelihood)模型是鉴定clade是否快速演化，

BUSTED (branch-site unrestricted statistical test for episodic diversification)检验某个基因在特定预设分支上是否经历了选择。

BUSTED (Branch-Site Unrestricted Statistical Test for Episodic Diversification) will test for gene-wide selection at pre-specified lineages. This method is particularly useful for relatively small datasets (fewer than 10 taxa) where other methods may not have sufficient power to detect selection. This method is not suitable for identifying specific sites subject to positive selection 资料提到BUSTED表现比PAML的branch-site模型表现更好。

如果需要 执行 episodic selection at individual sites

MEME (Mixed Effects Model of Evolution) tests for episodic selection at individual sites. Note that MEME does not accept a priori branch specifications (this feature is being introduced with v2.3-dev and later). MEME is the preferred approach for detecting positive selection at individual sites.

总的来说，HYPHY和PAML都是常用的分析软件，我自己对PAML更熟悉一点，因为之前在推断分化时间和进行kaks分析时候，我仔细阅读过PAML的一些学习资料，PAML的输入格式严格，我曾经因为基因名字太长，导致提取输出结果和基因名字之间的空格不存在了，然后一直报错，最后用KaKs_calculator完成交付之后，再仔细研究PAML的输出才发现。而且PAML在推断分化时间和计算dn/ds要求的格式又不一样，有一个要求的比较松弛,hyphy可能会好很多。
