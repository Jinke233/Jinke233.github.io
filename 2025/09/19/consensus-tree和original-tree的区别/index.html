consensus_tree和original tree的区别

一句话实用总结：

几乎都使用original tree，其上的bootstrap是从consensus tree上映射来的。

在系统发育分析中，Consensus Tree（共识树） 和 Original Tree（原始树） 是两种呈现结果的方式，它们各有侧重，相互补充。下面这个表格汇总了它们的核心区别，帮你快速建立整体认识：

特征 Original Tree (原始树) Consensus Tree (共识树)
定义 基于原始序列数据，通过特定算法（如NJ、ML）直接推导出的最优系统发育树。 通过Bootstrap重抽样产生大量重复数据并建树，汇总这些树的结果得到一致拓扑结构。
核心信息 包含分支长度（直观反映遗传距离）和拓扑结构。 主要反映拓扑结构的稳定性，不包含可靠的分支长度信息。
关键数值 可能展示从Bootstrap Consensus树映射过来的支持率。 在分支节点处标注Bootstrap支持率（或类似置信值）。
主要目的 展示基于原始数据推断出的最佳进化关系和遗传距离。 评估系统发育树中各分支节点的可信度/稳定性。
关系 是Bootstrap分析中众多可能树中的一棵。 是综合了许多Bootstrap重复树（通常包括Original Tree）信息后得出的一致意见。

🧬 1. Original Tree（原始树）：最佳估计

Original Tree 是使用完整的原始数据集，通过选定的系统发育推断方法（如邻接法 Neighbor-Joining, NJ；最大似然法 Maximum Likelihood, ML；最大简约法 Maximum Parsimony, MP 等）构建出的一棵最优树。它提供了：

  • 分支长度（Branch Length）：直观表示遗传距离的差异，例如序列替换率。
  • 拓扑结构（Topology）：展示了物种或基因之间的分组关系。 在MEGA等软件中，Original Tree 有时也会在分支上显示支持率，但这个支持率通常是从Bootstrap分析中映射过来的，并非其自身计算产生。

🔄 2. Consensus Tree（共识树）：稳定性的度量

Consensus Tree 并不是直接从原始数据计算出来的，而是通过统计重抽样方法（最常用的是Bootstrap法）来评估系统发育树中各个分支的置信度。

  • Bootstrap过程：从原始数据中有放回地随机抽取与原始数据量相同的位点，生成一个新的重抽样数据集。用同样的方法构建一棵系统发育树。将此过程重复成百上千次（如1000次），得到许多Bootstrap树。
  • 构建共识树：分析所有这些Bootstrap树，统计所有或多数树中都出现的分支拓扑结构。在这些共同出现的分支节点上，会标注一个Bootstrap支持率（Bootstrap support value），表示该分支在重抽样数据集中出现的频率（例如，1000次中有950次都出现了这个分支，支持率就是95%）。 因此，Consensus Tree 核心作用是评估Original Tree（或其他方法得出的树）中各个分支的可靠性和稳定性。一个高的Bootstrap值（通常认为≥70%或95%）意味着该分支在多次重抽样中都很稳定，结果更可靠。它通常不包含有实际生物学意义的分支长度信息。

💡 3. 如何结合理解与使用？

在实际研究和解读系统发育树时，需要将Original Tree和Consensus Tree的信息结合起来：

  1. 关注高支持率的分支：在Consensus Tree上，Bootstrap支持率高的分支（如≥70%或95%），我们认为其在Original Tree中的对应分支具有较高的置信度，进化关系更可靠。
  2. 谨慎解读低支持率分支：对于支持率较低的分支（如<50%），即使它在Original Tree中存在，也应持谨慎态度。这表明数据可能不足以强有力地支持该分组关系，其拓扑结构不确定性较大。
  3. 综合解读：一篇严谨的研究论文在展示系统发育树时，通常会在一棵树上同时显示分支长度（来自Original Tree）和Bootstrap支持率（来自Consensus Tree）。这样既能展示进化距离的估计，也能表明各个关系的置信度。

核心要点总结：

  • Original Tree告诉你“最佳进化路径是什么”，它提供了包括分支长度在内的具体进化关系假设。
  • Consensus Tree告诉你“这个进化路径的哪些部分比较可靠”，它通过Bootstrap支持率量化了拓扑结构的稳定性。
  • 二者相辅相成，缺一不可。没有支持率的Original Tree结论是脆弱的，而没有Original Tree具体拓扑和分支长度的Consensus树则无法完整描述进化关系。
