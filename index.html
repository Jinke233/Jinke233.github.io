不完全谱系分选和趋同演化的讨论

翻译中文

不完全谱系分选（ILS）能够导致相同基因型在系统发育树不同分支的物种间随机分布，且不受物种形成顺序的影响。需要特别强调的是，ILS（或称半同塑性）与趋同演化（同塑性）存在本质区别——后者同样能够导致远缘谱系间产生形态相似性（Darwin, 1859; Muschick et al., 2012; Sackton and Clark, 2019; Stern, 2013; Sun et al., 2018）。两者的关键差异在于：趋同演化通过不同（且通常未知的）遗传基础独立产生相似表型，而ILS则是由于并系遗传保留了相同等位基因，从而导致表型相似。

趋同演化作为解释非近缘类群形态相似性的机制已被广泛认可，而ILS的表型效应却长期被忽视，这很可能是因为此前存在难以克服的方法学挑战。然而，在经历快速物种形成的谱系中，忽视ILS可能导致对表型演化历史的错误解读。具体而言：
1）趋同演化通常需要持续的正向选择驱动；
2）ILS则要求祖先群体具有较大的有效种群规模（Ne），且在后续物种形成过程中未受强烈定向选择；
3）一旦生殖隔离和物种形成完成，跨物种保留的ILS区域就不再需要强自然选择的作用。

虽然部分ILS区域可能在分化后的谱系中受到选择，但此时ILS特征已在基因组中不可逆地固定。因此，区分非姐妹分类群间形态相似性源于趋同演化还是ILS，构成了演化生物学的重要挑战，我们的概念验证研究也凸显了这一研究方向的关键性。

尽管不能完全排除趋同演化对某些非姐妹有袋类分支共享的特定形态特征的影响，但ILS的解释显然更具说服力，因为趋同演化绝不会导致全基因组范围的冲突信号。需要指出的是，我们的结论虽然具有重要价值，但目前仅基于有限数量的有袋类基因组数据。未来需要在更广泛的有袋类物种中分析更多形态特征，以进一步阐明ILS对有袋类系统发育的影响。

EndnoteX9 to X21

此次任务共花费2天时间，遂记录：

我粗略检索发现X21比X9更新很多功能，本着体验新功能的想法还是决定更新。

更新时候有很多坑，这里记录。

我首先犯得毛病还是和之前一样不知悔改，拿到一款软件遇到问题无脑检索中文资料，结果一团糟，因为中文优质解决方案很少，来来回回都是简单的重复。

我首先遇到的问题是X21的library无法从X9转换。因为X9.2我用的是破解版，我怀疑是破解版问题。经过反复安装尝试，我发现破解版之后的Endnote界面确实有问题，只显示10个文献，我选择试用时候，以为是破解版问题，然而今早我发现试用版，endnote本来就显示10个文献。这一点和之前的一些软件有差别，我潜意识觉得既然你让我试用，就应该开放所有功能给我，我觉得好了我才买，结果我试用时候你还抠抠索索。这样那就不是试用版的问题，就是转换本身有毛病。

经过思考，通过云同步来解决。我一开始无脑同步，最后提示我2GB云空间附件限制。我查阅资料发现EndnoteX9之后桌面版激活之后，Endnote online附件是无限制的。但是我明明通过不同方式确实激活了桌面版Endnote,但是仍然只有2GB，我百思不得其解。本来我打算注册两个邮箱账号，切割我的原始文库为2份文库，同步到2个账户上面。我后面灵机一动，把我的X9库共享给X21不就好了，我邀请了我的另一个邮箱，这时我用学校购买的Endnote21 填写我的另一个邮箱账号，发现还是无法检索到贡献的文献，卒！

这时我灵机一动，在X21又查看同步信息，发现这次更换邮箱之后反而附件无限制，然后我在X9又改为的X21账户，发现和官网说的一致，online附件无限制，这样就可以同步来解决了。至于为什么一开始的邮箱不可以，我也不是很理解，后面Endnote进阶还是需要自学，我们学校图书馆教的都是最基础的功能，我发现Endnote桌面版可以分析文献，我还不知道到底怎么共享。

另外我发现一个我自己的问题不止一次，我好像对解决软件BUG上瘾，全然不顾一开始的学术目标，下次我要积极改正。不过Endnote是我用的一直以来的生产力工具，两天时间又了解了一些我还是比较宽慰的。

Orthofinder和OrthoMCL的区别

我自己不知道写的对不对，但是之前已经有两个老师问过我这个问题，但是他们也没有给我明确的答复。

粗略的答案就是orthofinder 一键输入自动输出orthologs,orthogroups，gene tree,species tree等文件。

但是我觉得精细的答案是orthofinder之所以推断的orthologroups更准确，是因为在第一个版本的orthofinder（OrthoFinder: solving fundamental biases in whole genome comparisons dramatically improves orthogroup inference accuracy）

就详细阐述了基于基因长度和系统发育距离对BLAST hits得分进行标准化，再用这些标准化的bit score得到RBNH(Reciporal best length normalized hit),再基于RNBH得到一个初步的同源组基因序列的阈值，最后MCL算法聚类得到直系同源组。

orthofinder之所以后续结果准确率高都是因为前期充分考虑了基因长度对鉴定相似基因造成的误差，不知道我说的对不对。

hyphy_note

上个星期和一位老师交流，老师问我有没有用过hyphy这个软件，我当时说不知道，所以赶紧回来学习一下。

Hyphy - Hypothesis testing using Phylogenies

Hyphy is an open-source software package for the analysis of genetic sequences using techniques in phylogenetics, molecular evolution, and machine learning. It features a complete graphical user interface (GUI) and a rich scripting language for limitless customization of analyses. Additionally, Hyphy features support for parallel computing environments (via message passing interface (MPI)) and it can be complied as a shared library and called from other programming environments such as Python and R. Hyphy is the computational backbone powering datamonkey.org.

虽然github介绍有GUI界面但是GUI界面版本的Hyphy已经不在维护，要么使用CLI，或者使用在线网站datamonkey。

首先正选择分析一般使用PAML，Easycodeml,hyphy分析.

以PAML为例 会使用三种模型

site 模型，branch模型,branch-site模型，我刚大致看了几篇文章，和预期的类似一般会选择branch-site模型。之所以不简单使用dn/ds计算是因为这种方式十分严格。

某些文献称：

branch模型鉴定出来的基因成为快速进化基因

branch-site模型鉴定出来的基因成为正选择基因

但是我发现hyphy的aBSREL(adaptive brach-site random effects likelihood)模型是鉴定clade是否快速演化，

BUSTED (branch-site unrestricted statistical test for episodic diversification)检验某个基因在特定预设分支上是否经历了选择。

BUSTED (Branch-Site Unrestricted Statistical Test for Episodic Diversification) will test for gene-wide selection at pre-specified lineages. This method is particularly useful for relatively small datasets (fewer than 10 taxa) where other methods may not have sufficient power to detect selection. This method is not suitable for identifying specific sites subject to positive selection 资料提到BUSTED表现比PAML的branch-site模型表现更好。

如果需要 执行 episodic selection at individual sites

MEME (Mixed Effects Model of Evolution) tests for episodic selection at individual sites. Note that MEME does not accept a priori branch specifications (this feature is being introduced with v2.3-dev and later). MEME is the preferred approach for detecting positive selection at individual sites.

总的来说，HYPHY和PAML都是常用的分析软件，我自己对PAML更熟悉一点，因为之前在推断分化时间和进行kaks分析时候，我仔细阅读过PAML的一些学习资料，PAML的输入格式严格，我曾经因为基因名字太长，导致提取输出结果和基因名字之间的空格不存在了，然后一直报错，最后用KaKs_calculator完成交付之后，再仔细研究PAML的输出才发现。而且PAML在推断分化时间和计算dn/ds要求的格式又不一样，有一个要求的比较松弛,hyphy可能会好很多。
